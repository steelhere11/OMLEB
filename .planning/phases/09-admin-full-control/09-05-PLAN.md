# Plan 09-05: Admin Comments System

<objective>
Implement a comments system where admins can add general or per-equipment comments on reports, creating a feedback channel for documenting review notes and communicating with technicians.
</objective>

## Context

Plan 09-01 created the reporte_comentarios table, RLS policies, ReporteComentario type, and the table has admin full access + technician read-only policies.

### Comments Scope
- General comments: equipo_id = NULL, appear at the report level
- Per-equipment comments: equipo_id set, appear on specific equipment card

## Tasks

### Task 1: Create admin comments server actions
**File:** `src/app/actions/admin-comments.ts` (NEW)

```typescript
'use server'

export async function addAdminComment(
  reporteId: string,
  contenido: string,
  equipoId?: string
): Promise<{ success: boolean; error?: string }>

export async function deleteAdminComment(
  comentarioId: string
): Promise<{ success: boolean; error?: string }>
```

Both actions:
- Verify caller is admin role
- Use standard Supabase client (RLS handles access)
- addAdminComment: insert into reporte_comentarios with autor_id = current user
- deleteAdminComment: delete from reporte_comentarios
- Revalidate report paths

### Task 2: Create CommentSection component
**File:** `src/components/admin/comment-section.tsx` (NEW, ~140 LOC)

Props:
```typescript
interface CommentSectionProps {
  comments: Array<ReporteComentario & { autor_nombre?: string }>;
  reporteId: string;
  equipoId?: string; // if per-equipment comments
  equipos?: Array<{ id: string; etiqueta: string }>; // for scope selector
  readOnly?: boolean; // true for technician view
}
```

UI:
1. Section header: "Comentarios" with count badge
2. List of existing comments:
   - Author name and timestamp (formatted relative: "hace 2 horas" or date)
   - Comment text content
   - Delete button (trash icon, only shown if !readOnly)
   - If per-equipment: show equipment label tag
3. Add comment form (hidden if readOnly):
   - Textarea for content
   - If equipos provided and no equipoId: scope selector dropdown ("General" or specific equipment)
   - "Agregar comentario" submit button
   - Loading state
4. Empty state: "Sin comentarios" when no comments exist

All text in Spanish.

### Task 3: Integrate comments into report detail
**Files:**
- `src/app/admin/reportes/[reporteId]/page.tsx` (MODIFY) — fetch comments data
- `src/app/admin/reportes/[reporteId]/report-detail.tsx` (MODIFY) — render CommentSection

In page.tsx:
- Fetch reporte_comentarios for this report with author info (join with users table for nombre)
- Pass comments to report-detail component

In report-detail.tsx:
1. Import CommentSection
2. Add a general CommentSection at the bottom of the report detail (after all equipment sections)
   - Shows all general comments (equipo_id = NULL)
   - Provides equipos list for scope selector
3. Optionally: add per-equipment CommentSection inside each equipment card
   - Shows only comments for that equipment
   - equipoId pre-set, no scope selector needed
4. Wire up add/delete handlers to server actions

The general comment section at the bottom should be the primary comment area. Per-equipment comments are a nice-to-have if the layout supports it (check existing equipment card structure).
