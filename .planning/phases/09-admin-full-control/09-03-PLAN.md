# Plan 09-03: Photo Management — Delete, Flag, Upload from Admin

<objective>
Give admins full photo management capabilities: flag photos with review status (aceptada/rechazada/retomar), delete individual photos, and upload new photos to reports from the admin side.
</objective>

## Context

Plan 09-01 created the server actions (adminFlagPhoto, adminDeletePhoto, adminUploadPhoto) and the database columns (estatus_revision, nota_admin on reporte_fotos). This plan builds the UI components and integrates them into the admin report detail page.

### Current report-detail.tsx Photo Display
The admin report detail page currently shows photos in read-only grids. Check how photos are currently rendered — likely via a PhotoGrid or direct mapping of reporte_fotos. We need to replace/enhance this with interactive admin controls.

## Tasks

### Task 1: Create AdminPhotoCard component
**File:** `src/components/admin/admin-photo-card.tsx` (NEW, ~150 LOC)

A card that wraps each photo with admin management actions.

Props:
```typescript
interface AdminPhotoCardProps {
  foto: ReporteFoto;
  onFlag: (fotoId: string, estatus: FotoEstatusRevision, nota?: string) => Promise<void>;
  onDelete: (fotoId: string) => Promise<void>;
}
```

UI:
1. Photo thumbnail (clickable for lightbox/full-size view)
2. Status badge in corner:
   - pendiente: gray badge "Pendiente"
   - aceptada: green badge "Aceptada"
   - rechazada: red badge "Rechazada"
   - retomar: yellow/amber badge "Retomar"
3. Color-coded border matching status (green/red/yellow/gray)
4. Status dropdown to change flag (select with 4 options)
5. Note textarea (appears when changing to rechazada or retomar — admin explains why)
6. Delete button (small trash icon) with confirmation dialog ("Eliminar esta foto permanentemente?")
7. Show etiqueta label and metadata (GPS, date) below photo
8. If nota_admin exists, show it below the photo

All text in Spanish.

### Task 2: Create AdminPhotoUpload component
**File:** `src/components/admin/admin-photo-upload.tsx` (NEW, ~120 LOC)

Allows admin to upload a photo to a specific report.

Props:
```typescript
interface AdminPhotoUploadProps {
  reporteId: string;
  equipos: Array<{ id: string; etiqueta: string }>; // for equipo_id dropdown
  pasos?: Array<{ id: string; nombre: string }>; // for reporte_paso_id dropdown
}
```

UI:
1. "Agregar foto" button that expands to upload form
2. File input (accept="image/*,video/*")
3. Equipo dropdown (optional — "General" if no equipo selected)
4. Etiqueta dropdown (antes, durante, despues, dano, placa, progreso)
5. Paso dropdown (optional — only if pasos provided)
6. Upload button with loading state
7. Preview of selected image before upload
8. Uses adminUploadPhoto server action
9. On success: closes form and photo appears in grid
