# Plan 09-04: Edit Workflow Steps + Equipment Info from Report Detail

<objective>
Let admins edit workflow step data (readings, notes, completion status) and equipment info (marca, modelo, serie, tipo, Phase 8 nameplate fields) directly from the report detail page, without navigating away.
</objective>

## Context

Plan 09-01 created server actions: adminUpdateStep and adminUpdateEquipmentInfo. This plan builds the inline editor UI components and integrates them into the admin report detail page.

### Current Equipment/Step Display in report-detail.tsx
- Equipment entries show equipment info header + step details
- Steps show readings (if any), notes, and completion status as read-only
- Equipment header shows marca, modelo, etc. from equipos table
- Admin can already edit some reporte_equipo fields (tipo_trabajo, diagnostico, etc.)

### Equipment Fields Available for Editing
From equipos table: marca, modelo, numero_serie, tipo_equipo_id
From Phase 8 additions: capacidad, refrigerante, voltaje, fase, ubicacion

### Constants Available
- `src/lib/constants/nameplate-options.ts` — refrigerantes, voltajes options
- `src/lib/constants/ubicaciones.ts` — ubicacion dropdown options (ATM, PATIO, BOVEDA, TREN DE CAJA, OTRO)

## Tasks

### Task 1: Create AdminStepEditor component
**File:** `src/components/admin/admin-step-editor.tsx` (NEW, ~180 LOC)

Inline editor for a workflow step's data.

Props:
```typescript
interface AdminStepEditorProps {
  paso: ReportePaso; // current step data
  lecturasDef?: LecturaRequerida[]; // reading definitions (names, units, ranges)
  onSave: (data: { lecturas?: Record<string, any>; notas?: string; completado?: boolean }) => Promise<void>;
  onCancel: () => void;
}
```

UI:
1. Appears inline when admin clicks "Editar" on a step row
2. Shows all lecturas as editable inputs:
   - Text/numero: text input pre-filled with current value
   - Si/No: toggle switch
   - Numeric: number input with unit label and range hint
3. Notas: editable textarea pre-filled with current value
4. Completado: toggle with label
5. "Guardar" and "Cancelar" buttons
6. Loading state on save
7. Validation: numeric fields validate against ranges (yellow warning, not blocking)

### Task 2: Create AdminEquipmentInfoEditor component
**File:** `src/components/admin/admin-equipment-info-editor.tsx` (NEW, ~130 LOC)

Inline editor for equipment's master data.

Props:
```typescript
interface AdminEquipmentInfoEditorProps {
  equipo: Equipo; // current equipment data
  tiposEquipo: TipoEquipo[]; // for tipo dropdown
  onSave: (data: Partial<Equipo>) => Promise<void>;
  onCancel: () => void;
}
```

UI:
1. Inline form that appears when admin clicks "Editar equipo" on equipment card header
2. Fields:
   - marca: text input
   - modelo: text input
   - numero_serie: text input
   - tipo_equipo_id: dropdown populated from tiposEquipo
   - capacidad: text input
   - refrigerante: dropdown from nameplate-options.ts
   - voltaje: dropdown from nameplate-options.ts
   - fase: text input
   - ubicacion: dropdown from ubicaciones.ts
3. All fields pre-filled with current values
4. "Guardar" and "Cancelar" buttons
5. Saves directly to equipos table via adminUpdateEquipmentInfo

### Task 3: Integrate editors into report-detail.tsx
**File:** `src/app/admin/reportes/[reporteId]/report-detail.tsx` (MODIFY)

Read the current file first. Then:

1. Import AdminStepEditor and AdminEquipmentInfoEditor
2. For each step row in the report detail:
   - Add an "Editar" button (pencil icon or text link)
   - On click: toggle that step's row to show AdminStepEditor inline
   - Track editing state: `editingStepId` in component state
   - Only one step editable at a time
3. For each equipment card header:
   - Add "Editar equipo" link next to existing edit controls
   - On click: show AdminEquipmentInfoEditor inline below the header
   - Track editing state: `editingEquipoId` in component state
4. Wire save handlers to call the server actions from reportes.ts
5. On successful save: close editor, refresh data

Need to also fetch tipos_equipo data for the equipment editor dropdown. Check how the report detail page currently fetches data — add tipos_equipo to the query if not already present.
