     a. **Unique constraint** on reportes(folio_id, fecha) ŌĆö prevents duplicate daily reports:
     ```sql
     ALTER TABLE public.reportes
     ADD CONSTRAINT unique_folio_fecha UNIQUE (folio_id, fecha);
     ```

     b. **Fix folio_asignados RLS** ŌĆö let technicians see ALL team members for their assigned folios (not just their own row). Drop the existing restrictive policy and replace it:
     ```sql
     DROP POLICY IF EXISTS "folio_asignados_tech_select" ON public.folio_asignados;
     CREATE POLICY "folio_asignados_tech_select"
       ON public.folio_asignados FOR SELECT
       TO authenticated
       USING (folio_id IN (SELECT private.get_my_folio_ids()));
     ```
     NOTE: Check `supabase/rls.sql` for whether `private.get_my_folio_ids()` already exists. If not, create it as a SECURITY DEFINER function that returns folio IDs where the user is assigned. Follow the exact same pattern used by existing `private.*` helper functions in rls.sql.

     c. **Enable Supabase Realtime** for report tables:
     ```sql
     ALTER PUBLICATION supabase_realtime ADD TABLE public.reportes;
     ALTER PUBLICATION supabase_realtime ADD TABLE public.reporte_equipos;
     ALTER PUBLICATION supabase_realtime ADD TABLE public.reporte_materiales;
     ```

     Add a header comment explaining this file must be run AFTER schema.sql and rls.sql.

  1. Create `src/app/actions/reportes.ts` with "use server" directive. Import createClient from @/lib/supabase/server, validation schemas from @/lib/validations/reportes, ActionState from @/types/actions. Follow the exact same patterns as equipos.ts and folios.ts (role check, Zod validate, DB operation, revalidatePath).

     Implement these server actions:

     a. **getOrCreateTodayReport(folioId: string)**: Returns `{ reporteId: string } | { error: string }`.
        - Get authenticated user. Verify user exists.
        - Compute today's date as `new Date().toISOString().split("T")[0]`.
        - Query `reportes` for existing report matching (folio_id, fecha=today) using `.maybeSingle()`.
        - If exists, return `{ reporteId: existing.id }`.
        - If not, get folio's sucursal_id, insert new report (folio_id, creado_por=user.id, sucursal_id, fecha=today, estatus="en_progreso").
        - Handle 23505 (unique_violation) race condition: if insert fails with this code, retry the SELECT and return that report's ID.
        - After creating a new report, call `preFillFromPreviousReport()` (see below).
        - Return `{ reporteId: newReport.id }`.

     b. **preFillFromPreviousReport(supabase, folioId, newReporteId, today)**: Private async function (not exported).
        - Find the most recent previous report for this folio (fecha < today, order desc, limit 1).
        - If found, copy its reporte_equipos entries (equipo_id + tipo_trabajo only, NOT diagnostico/trabajo_realizado/observaciones ŌĆö those start fresh each day).
        - Insert the copied entries into reporte_equipos with the new reporte_id.

     c. **saveEquipmentEntry(reporteId: string, entryId: string | null, _prevState: ActionState | null, formData: FormData)**: Returns ActionState.
        - Auth check (user must exist, no admin-only restriction ŌĆö technicians and helpers can save).
        - Validate formData with reporteEquipoSchema.
        - If entryId is null: INSERT into reporte_equipos (reporte_id, equipo_id, tipo_trabajo, diagnostico, trabajo_realizado, observaciones). Empty strings become null.
        - If entryId is provided: UPDATE the existing reporte_equipos row by id.
        - revalidatePath for the report page.
        - Return `{ success: true, message: "Guardado" }`.

     d. **removeEquipmentEntry(entryId: string)**: Returns ActionState.
        - Auth check.
        - DELETE from reporte_equipos where id = entryId.
        - revalidatePath.
        - Return success.

     e. **saveMaterials(reporteId: string, materials: { cantidad: number; unidad: string; descripcion: string }[])**: Returns ActionState.
        - Auth check.
        - Validate each material with reporteMaterialSchema.
        - Delete all existing reporte_materiales for this reporteId (delete-all + re-insert pattern, same as folio_asignados).
        - Insert all valid materials.
        - revalidatePath.
        - Return success.

     f. **updateReportStatus(reporteId: string, _prevState: ActionState | null, formData: FormData)**: Returns ActionState.
        - Auth check.
        - Validate with reporteStatusSchema.
        - If estatus is "completado", verify at least one equipment entry exists (SELECT count from reporte_equipos). If none, return error: "No se puede completar un reporte sin entradas de equipo".
        - UPDATE reportes.estatus.
        - Also UPDATE the parent folio's estatus to match (fetch folio_id from the report, then update folios). Mapping: completado->completado, en_espera->en_espera, en_progreso->en_progreso.
        - revalidatePath for both the report page and /tecnico (folio list).
        - Return success.

     - Return `{ success: true, message: "Equipo agregado", data: { id: equipo.id } }` ŌĆö the data field lets the UI immediately add this equipment to the report.
     - revalidatePath for the report page (pass sucursal_id context if available).
