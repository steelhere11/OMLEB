  **Props:**
  ```typescript
  interface MaterialsSectionProps {
    reporteId: string;
    initialMaterials: ReporteMaterial[];
    isCompleted: boolean;
  }
  ```

  **State management:**
  - `rows` state: array of `{ id: string; cantidad: string; unidad: string; descripcion: string }`.
  - Initialize from initialMaterials (map DB records to form state with string values for controlled inputs). Use `crypto.randomUUID()` for client-side row IDs.
  - If initialMaterials is empty, start with one empty row so the form isn't blank.
  - `saved` state: boolean to show "Guardado" feedback.
  - useTransition for non-blocking save.

  **Row operations:**
  - `addRow()`: Append a new empty row with `crypto.randomUUID()` ID.
  - `removeRow(id)`: Filter out the row. Prevent removing the last row (always keep at least one).
  - `updateRow(id, field, value)`: Update a specific field in a specific row.

  **Save:**
  - "Guardar materiales" button at the bottom.
  - On press: filter out rows where ALL fields are empty (skip blank rows). Validate remaining rows: cantidad must be a positive number, unidad and descripcion must be non-empty. Show inline validation message if invalid.
  - Call `saveMaterials(reporteId, validRows)` with the cleaned data (cantidad parsed to number).
  - On success, show "Materiales guardados" indicator for 3 seconds, then fade.
  - On error, show the error message.

  **UI layout (mobile-first):**
  - Section header: "Materiales Utilizados" with material count badge.
  - Each row renders as a card-like block (not a table — tables are hard on mobile):
    - Row number indicator (1, 2, 3...)
    - cantidad: small Input (type="number", inputMode="decimal", placeholder "Cant.", width ~w-20)
    - unidad: Input (placeholder "Unidad (pza, m, kg...)", width ~w-28). Consider using a Select or datalist with common units: "pza", "m", "kg", "lt", "rollo", "tramo", "juego".
    - descripcion: Input (placeholder "Descripcion del material", flex-1, full width on its own line if needed)
    - Remove button: small red X or trash icon, right-aligned

  - On small screens, stack cantidad + unidad on one line, descripcion on the next line (responsive flex-wrap).
  - "+ Agregar material" button below the rows (secondary variant, full width).
  - "Guardar materiales" button (primary variant, full width) below the add button.

  If isCompleted, disable all inputs and hide add/remove/save buttons.

  **Common units datalist** (optional but helpful): Provide a `<datalist>` with id="unidades-comunes" listing: pza, m, kg, lt, rollo, tramo, juego, caja, par, bolsa. Attach to the unidad input.
