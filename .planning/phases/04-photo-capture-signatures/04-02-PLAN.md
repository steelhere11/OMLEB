1. Create `src/app/actions/fotos.ts` ("use server"):
   - `getPhotosForReport(reporteId: string)`: Fetches all `reporte_fotos` rows for the given reporte_id, ordered by created_at asc. Returns array of `ReporteFoto`.
   - `getPhotosForStep(reportePasoId: string)`: Fetches `reporte_fotos` rows where `reporte_paso_id` matches, ordered by created_at asc. Returns array of `ReporteFoto`.
   - `getPhotosForEquipment(reporteId: string, equipoId: string)`: Fetches `reporte_fotos` rows where `reporte_id` matches AND `equipo_id` matches AND `reporte_paso_id IS NULL` (general equipment photos, not step-specific). Returns array of `ReporteFoto`.
   - `deletePhotoAction(fotoId: string)`: Deletes the reporte_fotos row and the corresponding file from Supabase Storage. To get the file path for Storage deletion: parse the URL from the row's `url` field to extract the path after `/reportes/`. Use `createClient` from `@/lib/supabase/server` for all these. Revalidate `/tecnico`.

2. Create `src/components/shared/photo-thumbnail.tsx` ("use client"):
   - Props: `{ foto: ReporteFoto; onDelete?: (fotoId: string) => void; disabled?: boolean }`.
   - Renders a 64x64px (w-16 h-16) rounded-lg thumbnail with `object-fit: cover`.
   - Label badge: small colored pill in bottom-left corner showing the etiqueta (uses same etapaColors mapping: antes=blue, durante=amber, despues=green, dano=red, placa=gray, progreso=purple).
   - On tap: open a fullscreen lightbox overlay (fixed inset-0 z-50 bg-black/90) showing the full-size image centered with `object-fit: contain`.
   - Lightbox has X close button (top-right) and delete button (bottom center, red, "Eliminar foto") if onDelete is provided and not disabled.
   - Delete shows confirmation: "Eliminar esta foto?" with "Cancelar" / "Eliminar" buttons.

3. Create `src/components/shared/photo-gallery-row.tsx` ("use client"):
   - Props: `{ photos: ReporteFoto[]; onDelete?: (fotoId: string) => void; disabled?: boolean }`.
   - Renders a horizontal scrollable row (`overflow-x-auto`, `flex`, `gap-2`, `pb-2` for scroll thumb space).
   - Maps over photos and renders `PhotoThumbnail` for each.
   - If no photos, renders nothing (empty fragment).
   - Snap scrolling on mobile: `scroll-snap-type: x mandatory` on container, `scroll-snap-align: start` on each thumbnail.
