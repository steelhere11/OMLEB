<task type="auto">
  <name>Task 2: Create GPS utility, photo stamper, and photo uploader libraries</name>
  <files>
    src/lib/gps.ts
    src/lib/photo-stamper.ts
    src/lib/photo-uploader.ts
  </files>
  <action>
1. Create `src/lib/gps.ts`:
   - Export `getGpsPosition()` that returns `Promise<{ lat: number; lng: number; approximate: boolean } | null>`.
   - Use `navigator.geolocation.getCurrentPosition` with `enableHighAccuracy: true`, `timeout: 5000`, `maximumAge: 30000`.
   - Cache last known position in module-level variable. On error, return last known position with `approximate: true`. If no last known position, return `null`.
   - Handle PermissionDeniedError, TimeoutError, PositionUnavailableError gracefully -- never throw.
   - Export type `GpsPosition = { lat: number; lng: number; approximate: boolean }`.

2. Create `src/lib/photo-stamper.ts`:
   - Export `drawOverlayBadge(ctx, data, canvasWidth, canvasHeight)`.
   - `data` interface: `{ lat: number | null; lng: number | null; approximate: boolean; timestamp: Date }`.
   - Renders a compact semi-transparent black rounded rect badge in bottom-right corner.
   - Two lines of white monospace text: line 1 = date + time (es-MX locale, DD/MM/YYYY HH:MM:SS), line 2 = GPS coords (6 decimal places) or "GPS no disponible". If approximate, append " ~" to coords.
   - Font size 14px bold monospace. Padding 12px. Line height 18px. Badge offset 10px from bottom-right.
   - Use `ctx.roundRect()` for rounded corners (radius 6).
   - Follow the exact implementation from 04-RESEARCH.md Code Examples section.

3. Create `src/lib/photo-uploader.ts`:
   - Export `compressAndUpload(imageBlob, metadata, onProgress?)` returning `Promise<UploadResult>`.
   - `UploadResult` = `{ success: true; url: string; fotoId: string } | { success: false; error: string }`.
   - `metadata` shape: `{ reporteId: string; equipoId: string | null; reportePasoId: string | null; etiqueta: string; gps: string | null; fecha: Date }`.
   - Steps: (a) Convert Blob to File (`new File([blob], "photo.jpg", { type: "image/jpeg" })`), (b) compress with `browser-image-compression` (maxSizeMB: 0.8, maxWidthOrHeight: 1920, useWebWorker: true), (c) upload to Supabase Storage bucket `reportes` at path `{reporteId}/{equipoId|general}/{uuid}.jpg`, (d) get public URL, (e) insert row into `reporte_fotos` table with all metadata, (f) if DB insert fails, clean up uploaded file.
   - Use `createClient()` from `@/lib/supabase/client` (browser client, NOT admin/service role).
   - `onProgress` callback: 10% after start, 10-50% during compression, 50% before upload, 80% after upload, 100% after DB insert.
   - Export `deletePhoto(fotoId, filePath)` for removing a photo (delete from storage + delete from DB).

CRITICAL: `browser-image-compression` expects a `File` not a `Blob` -- always convert first. Follow research pattern exactly.
  </action>

    - `src/lib/photo-stamper.ts` exports `drawOverlayBadge`
    - `src/lib/photo-uploader.ts` exports `compressAndUpload` and `deletePhoto`

1. Create `src/components/shared/camera-capture.tsx`:
   - "use client" component. Props: `{ label: string; reporteId: string; equipoId: string | null; reportePasoId: string | null; onCapture: (result: { url: string; fotoId: string }) => void; onClose: () => void }`.
   - On mount: call `navigator.mediaDevices.getUserMedia({ video: { facingMode: { ideal: "environment" }, width: { ideal: 1920 }, height: { ideal: 1080 } }, audio: false })`.
   - Render a fullscreen fixed overlay (z-50, bg-black, inset-0).
   - Use two elements: a hidden `<video>` for the stream and a visible `<canvas>` that fills the screen.
   - In `requestAnimationFrame` loop: draw video frame to canvas, then call `drawOverlayBadge()` with current GPS + time.
   - Set canvas drawing buffer dimensions to match video stream resolution (NOT CSS size). Use CSS `object-fit: contain` on the canvas for layout.
   - On mount, also call `getGpsPosition()` and cache. Re-fetch GPS every 10 seconds.
   - Capture button: large white circular button at bottom center. On tap:
     (a) Draw final frame + overlay to canvas.
     (b) Call `canvas.toBlob(callback, "image/jpeg", 0.85)`.
     (c) Show small loading spinner on the capture button.
     (d) Call `compressAndUpload(blob, { reporteId, equipoId, reportePasoId, etiqueta: label, gps: formatted lat,lng string or null, fecha: new Date() }, onProgress)`.
     (e) On success: call `onCapture({ url, fotoId })` -- auto-accept behavior (no preview screen).
     (f) On error: show brief red toast/message at top, stay on camera for retry.
   - Close button: X icon in top-left corner. Calls `onClose()`.
   - Label badge: show the current label (e.g., "ANTES") in the top-right corner so tech knows what they're capturing.
   - Upload progress: small progress bar or percentage text near capture button while uploading.
   - CLEANUP: In useEffect cleanup, ALWAYS call `stream.getTracks().forEach(track => track.stop())`. Also cancel `requestAnimationFrame`.
   - Camera error handling: if getUserMedia fails (permission denied, no camera), show Spanish error message with close button. Never crash.
   - Lock to portrait: use fixed positioning + prevent scrolling. CSS only, no orientation API for camera.
