  <action>
Create `src/components/shared/signature-pad.tsx` ("use client"):

Props: `{ onSave: (data: { nombre: string; firma: string }) => void; onCancel: () => void }`.

Layout and behavior:
- Fullscreen fixed overlay (inset-0 z-50 bg-white) that takes over the entire screen.
- Use a `ref` container div for requesting fullscreen.
- On mount: attempt fullscreen (`containerRef.current.requestFullscreen()`) then landscape lock (`screen.orientation.lock("landscape")`). Both in try/catch -- if either fails, fall back gracefully.
- Track `isLandscape` state: `window.innerWidth > window.innerHeight`. Listen to `resize` event to update.
- If not landscape and lock failed, show a subtle "Gira tu telefono para firmar" hint with a rotation icon.

Structure (top to bottom in landscape):
1. Header bar: "Firma del Encargado" title + input field for `nombre` (placeholder: "Nombre del encargado de sucursal").
2. Canvas area: takes remaining space. White background. Black pen color.
3. Footer bar: "Borrar" button (secondary, clears canvas), "Cancelar" button (secondary, calls onCancel), "Guardar Firma" button (primary, validates and calls onSave).

Signature pad initialization:
- Import `SignaturePad` from `signature_pad` (vanilla library, not React wrapper).
- Create pad instance on canvas ref with `{ penColor: "rgb(0, 0, 0)", backgroundColor: "rgb(255, 255, 255)" }`.
- Handle high-DPI: set canvas width/height attributes to `offsetWidth * devicePixelRatio` / `offsetHeight * devicePixelRatio`, then scale context by devicePixelRatio.
- Handle resize: on window resize, save data -> resize canvas -> restore data (signature_pad has `toData()` / `fromData()` methods).

Save validation:
- Both `nombre` (trimmed, non-empty) and signature (not empty via `padRef.current.isEmpty()`) are required.
- If either missing, show inline Spanish validation message (e.g., "Ingresa el nombre del encargado" or "Se requiere la firma").
- On valid save: call `onSave({ nombre: nombre.trim(), firma: padRef.current.toDataURL("image/png") })`.

Cleanup:
- `padRef.current.off()` to detach event listeners.
- Exit fullscreen if active: `document.exitFullscreen()`.
- Unlock orientation: `screen.orientation.unlock()`.
- Remove resize listener.

IMPORTANT per CONTEXT.md decisions:
- Single "Borrar" button to clear -- no undo, just wipe and re-sign.
- Force landscape orientation when pad opens.
- Capture both typed name AND drawn signature.

2. **Update `src/app/actions/reportes.ts` `updateReportStatus` function**:
   - Current function reads `estatus` from formData. Extend it to also read `firma_encargado` and `nombre_encargado` from formData.
   - When status is "completado": update the `reportes` row with `estatus`, `firma_encargado`, AND `nombre_encargado` (all three in one update).
   - When status is NOT "completado": update only `estatus` (same as current behavior).
   - The firma_encargado value is a base64 data URL string (e.g., "data:image/png;base64,...") stored directly in the text column.
   - Keep all existing behavior: equipment count validation for completado, folio status sync, revalidation.

3. **Modify `src/app/tecnico/reporte/[reporteId]/status-section.tsx`**:
   - Add state: `showSignaturePad: boolean` (default false).
   - Add state: `signatureData: { nombre: string; firma: string } | null` (default null).
   - Current flow: tech selects status -> taps "Guardar y Enviar Reporte" -> form action fires.
   - New flow for Completado:
     (a) Tech selects "Completado" status and taps "Guardar y Enviar Reporte".
     (b) INSTEAD of immediately submitting the form, intercept: if `selectedStatus === "completado"` and `signatureData` is null, set `showSignaturePad = true` and prevent form submission.
     (c) SignaturePad opens fullscreen. Branch manager types name, draws signature, taps "Guardar Firma".
     (d) `onSave` callback stores the data in `signatureData` state and closes the pad.
     (e) Now the submit button text changes to "Enviar Reporte Completado" (green emphasis) and the section shows a confirmation: "Firma recibida de: {nombre}" with a small signature preview and a "Cambiar firma" link.
     (f) Tech taps the submit button again. This time signatureData exists, so the form submits with firma_encargado and nombre_encargado as hidden inputs alongside estatus.
   - For non-Completado statuses: behavior is unchanged. No signature required.
   - When `showSignaturePad` is true: render `<SignaturePad onSave={...} onCancel={...} />`.
   - onCancel: close signature pad, clear signatureData, stay on status section.
   - Add hidden inputs in the form for firma_encargado and nombre_encargado (populated from signatureData).
   - The form still uses `useActionState` with the existing `updateReportStatus` server action -- just now with additional hidden form fields.

IMPORTANT flow detail: The signature screen appears AFTER the tech taps submit with Completado selected, but BEFORE the actual server action fires. It's a client-side gate. The form only submits to the server after the signature is collected.

Button text changes:
- Default (no signature yet): "Guardar y Enviar Reporte"
- After signature collected with Completado selected: "Enviar Reporte Completado"
- Non-completado statuses always show: "Guardar y Enviar Reporte"
