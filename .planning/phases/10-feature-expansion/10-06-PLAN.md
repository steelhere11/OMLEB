# Plan 10-06: Custom "Otro" Step for All Equipment Types

## Objective

Allow technicians to add custom ad-hoc maintenance steps not covered by the predefined templates. The "Agregar paso personalizado" button appears at the bottom of every equipment's preventive workflow and supports full evidence capture.

## Current State

- `reporte_pasos.nombre_custom` column already exists (added in 10-01 migration)
- Steps with both `plantilla_paso_id=NULL` and `falla_correctiva_id=NULL` represent custom steps
- Workflow step card renders based on plantilla data — needs to handle null plantilla gracefully

## Tasks

### Task 1: Create addCustomStep server action

**File:** `src/app/actions/report-actions.ts` (or new file)

```ts
export async function addCustomStep(
  reporteEquipoId: string,
  nombre: string,
  procedimiento?: string
): Promise<ReportePaso>
```

Creates a `reporte_pasos` row with:
- `reporte_equipo_id` = provided
- `plantilla_paso_id` = NULL
- `falla_correctiva_id` = NULL
- `nombre_custom` = provided nombre
- `notas` = provided procedimiento (optional)
- `completado` = false
- `orden` = next available order number for this equipment entry

### Task 2: Create CustomStepForm component

**New file:** `src/components/tecnico/custom-step-form.tsx` (~100 LOC)

Inline expandable form at the bottom of the step list:
- "Agregar paso personalizado" button (dashed border, + icon)
- On click, expands to show:
  - Nombre del paso (required text input)
  - Descripcion/procedimiento (optional textarea)
  - "Agregar" and "Cancelar" buttons
- On submit, calls `addCustomStep` and optimistically adds the step to the list
- Form collapses after successful add

### Task 3: Integrate into preventive workflow

**File:** `src/app/tecnico/reporte/[reporteId]/workflow-preventive.tsx`

- Import and render `CustomStepForm` after the last step card
- Pass `reporteEquipoId` to the form
- When a custom step is added, refresh the step list to include it

### Task 4: Integrate into corrective workflow

**File:** `src/app/tecnico/reporte/[reporteId]/workflow-corrective.tsx`

- Same integration: add `CustomStepForm` after the corrective issue list
- For additional work not covered by the predefined fallas

### Task 5: Update workflow step card to handle custom steps

**File:** `src/components/tecnico/workflow-step-card.tsx`

When `plantilla_paso_id` is null and `nombre_custom` is set:
- Display `nombre_custom` as the step title instead of `plantilla.nombre`
- Display `notas` as the procedure description
- Still show evidence stages (antes/durante/despues) and completion toggle
- Mark visually as a custom step (subtle indicator)

### Task 6: Update PDF to render custom steps

**File:** `src/components/pdf/report-document.tsx`

Custom steps (where plantilla_paso_id is null) render under a "Pasos adicionales" subsection within the equipment block:
- Step name from `nombre_custom`
- Same StepBlock rendering (readings, photos, notes)
- Appears after all template-based steps

### Task 7: Admin editing support for custom steps

**File:** Admin step editor components

Ensure the inline step editor from Phase 9 can edit `nombre_custom` in addition to readings and notes.
