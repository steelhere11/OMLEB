# Plan 10-01: Database Migration + Types

## Objective

Create the database migration and TypeScript type updates for all Phase 10 features: revision tracking, custom steps, and photo annotation etiqueta.

## Tasks

### Task 1: Database migration for Phase 10 features

**File:** `supabase/migration-10-feature-expansion.sql`

New tables and columns:
1. `reporte_revisiones` table (revision audit trail) with: id, reporte_id, autor_id, numero, resumen, cambios (jsonb), created_at
2. `reportes.revision_actual` column (integer, default 0)
3. `reporte_pasos.nombre_custom` column (custom step name for ad-hoc steps)
4. Updated `reporte_fotos.etiqueta` CHECK constraint to include 'anotado'
5. RLS policies: admin full access, tech read-only on assigned folios

### Task 2: Update TypeScript types

**Files:** `src/types/index.ts`, `src/types/workflows.ts`

- Add `ReporteRevision` and `CambioRevision` interfaces
- Add `revision_actual` to `Reporte` type
- Add `nombre_custom` to `ReportePaso` type
- Add `'anotado'` to `FotoEtiqueta` type

## Files Modified

1. `supabase/migration-10-feature-expansion.sql` — New migration file

3. `src/types/workflows.ts` — nombre_custom on ReportePaso
4. `src/app/tecnico/reporte/[reporteId]/workflow-preventive.tsx` — Minor type reference update
