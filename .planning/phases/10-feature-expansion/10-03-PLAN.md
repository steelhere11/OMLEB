# Plan 10-03: PDF Comment-Photo Pairing

## Objective

Improve the PDF report layout so that step notes appear directly below step photos (not separated in a metadata area), and admin comments appear in relevant equipment sections of the PDF. This is primarily a **PDF layout + data pipeline change** — no database changes needed.

## Current State

- `report-document.tsx` (1301 lines) renders the PDF with `@react-pdf/renderer`
- `StepBlock` component renders: step header → readings table → notes → photos (in that order)
- Notes currently render as plain text `"Nota: {text}"` ABOVE photos (line 801-803)
- Admin comments (`reporte_comentarios`) are NOT included in the PDF at all
- Comments are fetched in admin `page.tsx` and passed to `report-detail.tsx`, but not to `ReportPdfButton`
- `ReporteComentario` type exists in `src/types/index.ts` with fields: id, reporte_id, equipo_id, autor_id, contenido, created_at

## Tasks

### Task 1: Reorder StepBlock — notes below photos

**File:** `src/components/pdf/report-document.tsx`

In the `StepBlock` component (line 745-811), reorder the content so photos come before notes:

Current order (completed step with content):
1. Step header
2. Readings table
3. Notes ← move this
4. Photos

New order:
1. Step header
2. Readings table
3. Photos ← now first
4. Notes ← now below photos with new styling

**Style change for notes:** Replace the plain `stepNote` style with a callout-style block:
- Italic text
- Left border accent (2px blue-ish left border)
- Slight left margin to align with photo grid indent
- Light background tint

Add new style `stepNoteCallout`:
```
stepNoteCallout: {
  fontSize: 8,
  fontStyle: 'italic',
  color: GRAY_700,
  marginLeft: 16,
  marginTop: 4,
  marginBottom: 4,
  paddingLeft: 8,
  paddingVertical: 3,
  borderLeftWidth: 2,
  borderLeftColor: BLUE,
  backgroundColor: '#f0f4ff',
}
```

Replace the notes rendering from:
```tsx
{step.notas && (
  <Text style={s.stepNote}>Nota: {step.notas}</Text>
)}
```
To:
```tsx
{step.notas && (
  <View style={s.stepNoteCallout}>
    <Text style={{ fontSize: 8, fontStyle: 'italic', color: GRAY_700 }}>{step.notas}</Text>
  </View>
)}
```

And move it AFTER the photos block.

### Task 3: Render equipment-scoped comments in PDF equipment sections

**File:** `src/components/pdf/report-document.tsx`

In the main `ReportDocument` component, within each equipment entry card (after the step blocks and orphan photos), render equipment-scoped comments:

1. Filter `data.comments` by `equipo_id` matching the current equipment entry
2. Need to match by equipment — the current equipment entries don't have an equipo_id. We need to add `equipo_id` to the equipment entry data type.

**Updated approach:** Add `equipo_id: string` to the equipment entry in `PdfReportData.equipmentEntries[].equipo`. Then in `ReportPdfButton`, pass through the equipo id.

For each equipment section, after orphan photos, render matching comments:
```tsx
{/* Admin comments for this equipment */}
{equipComments.length > 0 && (
  <View style={s.commentsSection}>
    <Text style={s.commentsTitle}>Comentarios del Administrador</Text>
    {equipComments.map((c, cIdx) => (
      <View key={cIdx} style={s.commentBlock}>
        <Text style={s.commentText}>{c.contenido}</Text>
        <Text style={s.commentMeta}>— {c.autorNombre}, {formatDate(c.created_at)}</Text>
      </View>
    ))}
  </View>
)}
```

New styles needed:
```
commentsSection: { paddingHorizontal: 8, paddingVertical: 6 }
commentsTitle: { fontSize: 8, fontWeight: 700, color: GRAY_500, textTransform: 'uppercase', letterSpacing: 0.4, marginBottom: 4 }
commentBlock: { marginBottom: 4, paddingLeft: 8, borderLeftWidth: 2, borderLeftColor: AMBER }
```
