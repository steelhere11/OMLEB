# Plan 10-02: Report Revision System

## Objective

Implement the report revision system that allows admins to edit approved reports with full audit trail. Each edit creates a numbered revision entry with summary and change details.

## Tasks

### Task 1: Server actions for revisions

**File:** `src/app/actions/admin-revisions.ts`

1. `createRevision(reporteId, resumen, cambios)` — insert revision row, increment revision_actual
2. `getRevisionHistory(reporteId)` — fetch all revisions with author names ordered by numero

### Task 2: Revision UI components and integration

**Files:**
- `src/components/admin/revision-history-panel.tsx` — Collapsible timeline showing all revisions with author, date, summary
- `src/app/admin/reportes/[reporteId]/report-detail.tsx` — Revision badge in header ("Revision X"), "Registrar revision" button for approved reports, revision summary modal
- `src/app/admin/reportes/[reporteId]/page.tsx` — Fetch revision data and pass to detail
- `src/components/admin/report-pdf-button.tsx` — Pass revision data to PDF

### Task 3: PDF revision display

**File:** `src/components/pdf/report-document.tsx`

- Show "Revision X" in header next to date when revision_actual > 0
- Show "Ultima revision: [fecha] por [admin]" in footer

### Task 4: Unlock approved reports for editing

**File:** `src/app/admin/reportes/[reporteId]/report-detail.tsx`

- Remove irreversible warning from approval action
- Allow post-approval edits that auto-create revision entries

## Files Modified

1. `src/app/actions/admin-revisions.ts` — New server actions file
2. `src/components/admin/revision-history-panel.tsx` — New revision timeline component
3. `src/app/admin/reportes/[reporteId]/report-detail.tsx` — Revision badge, edit button, summary modal
4. `src/app/admin/reportes/[reporteId]/page.tsx` — Fetch revision data
5. `src/components/admin/report-pdf-button.tsx` — Pass revision data
6. `src/components/pdf/report-document.tsx` — Revision in header/footer

## Verification

1. Admin can edit an approved report and each edit creates a revision entry
2. Report header shows "Revision X" badge when revision_actual > 0
3. Revision history panel shows timeline with author, date, summary
4. PDF includes revision number in header and last revision info in footer
5. Build passes
