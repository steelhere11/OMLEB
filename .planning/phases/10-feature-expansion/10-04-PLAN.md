# Plan 10-04: Photo Annotation Component

## Objective

Build a full-screen canvas-based photo annotation tool that lets technicians and admins draw freehand lines, arrows, and text labels on photos. The annotated image **overwrites the original** in Supabase Storage — no duplicate rows, no extra storage.

## Current State

- Photos are displayed via `AdminPhotoCard` (admin) and photo thumbnails (technician)
- Photos are stored in Supabase Storage with URLs in `reporte_fotos.url`
- The `etiqueta` constraint already includes 'anotado' (added in 10-01 migration)
- Etiqueta does NOT change when annotating — the photo keeps its original etiqueta

## Tasks

### Task 1: Create PhotoAnnotator component

**New file:** `src/components/shared/photo-annotator.tsx` (~300 LOC)

Full-screen modal canvas editor:
- Load image as canvas background via `drawImage()`
- Bottom toolbar: Draw (freehand), Text (tap to place), Arrow, Color picker (red/blue/white/yellow), Stroke width toggle (thin/thick), Undo, Save
- Touch-optimized with pointer events (pointerdown, pointermove, pointerup)
- Undo stack: each action (stroke, text, arrow) stored as array, undo pops last
- "Guardar" button: flatten canvas to JPEG blob via `canvas.toBlob()`
- "Cancelar" button: close without saving
- Landscape-friendly layout (like camera/signature components)

**Drawing modes:**
- **Freehand:** continuous path following pointer
- **Text:** tap location, prompt for text input, render on canvas
- **Arrow:** drag from start to end, render line with arrowhead

**Color presets:** `#FF0000` (red), `#0066FF` (blue), `#FFFFFF` (white), `#FFD600` (yellow)
**Stroke widths:** thin (2px), thick (5px)

### Task 2: Create overwriteAnnotatedPhoto server action

**New file or addition to:** `src/app/actions/photo-actions.ts`

```ts
export async function overwriteAnnotatedPhoto(fotoId: string, imageBlob: FormData): Promise<void>
```

Flow:
1. Get the current photo record from DB (get URL path)
2. Upload the new image to the SAME storage path (overwrite)
3. No DB row changes needed — same URL, same row
4. Revalidate the report page

### Task 3: Integrate "Anotar" button into AdminPhotoCard

**File:** `src/components/admin/admin-photo-card.tsx`

Add an "Anotar" button (pencil icon) to the photo card action buttons. On click, open the `PhotoAnnotator` modal with the photo URL. On save, call `overwriteAnnotatedPhoto` and refresh.

### Task 4: Integrate "Anotar" button into technician photo views

**Files:** Technician photo components (photo thumbnails in workflow steps)

Add the same "Anotar" button on technician-side photo thumbnails. Same flow: open annotator, save overwrites original.

### Task 5: Mobile viewport testing and polish

Ensure:
- Canvas fills screen properly on mobile
- Touch drawing works smoothly (no scroll interference)
- Toolbar buttons are large enough for thumb taps
- Text input modal doesn't block canvas
- Arrow drawing gives visual feedback during drag
