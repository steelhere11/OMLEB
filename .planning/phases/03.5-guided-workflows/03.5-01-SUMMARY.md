# Phase 3.5 Plan 01: Guided Maintenance Workflows Summary

**Structured step-by-step preventive workflows and corrective issue picker replacing free-text reporting — with real-time reading validation and auto-save**

## Performance

- **Duration:** 8 min
- **Tasks:** 4
- **Files created:** 6
- **Files modified:** 8

## Accomplishments
- 7 workflow server actions (getWorkflowTemplates, getCorrectiveIssues, getTiposEquipo, getStepProgress, saveStepProgress, saveCorrectiveSelection, getValoresReferencia)
- ReadingInput component handling text, Sí/No toggle, and numeric modes with real-time range validation
- WorkflowStepCard with expandable accordion showing procedure, photo placeholders, reading inputs, notes, and completion toggle
- WorkflowPreventive container loading templates from plantillas_pasos with progress bar and auto-save
- WorkflowCorrective container loading issues from fallas_correctivas with multi-select picker and detail cards
- CorrectiveIssuePicker with tappable cards, checkboxes, and diagnostic preview
- Equipment entry form integration: conditional render based on tipo_trabajo (preventivo/correctivo)
- Add-equipment modal updated with tipos_equipo dropdown replacing free-text input
- Fallback to free-text textareas for equipment type "otro" or missing templates
- Type updates: tipo_equipo_id on Equipo, FotoEtiqueta with 'durante', workflow type re-exports

## Decisions Made

| Decision | Rationale |
|----------|-----------|
| Auto-save on step completion | Techs spend extended time on each step; granular saves prevent data loss without requiring explicit save buttons |
| Yellow warning for range validation | Non-blocking because techs know their equipment better than generic ranges; warnings inform but don't obstruct |
| Fallback to free-text for "otro" | Equipment without matching templates still needs reporting capability; graceful degradation |
| Photo buttons as placeholders | Phase 4 handles camera integration; buttons show what evidence is needed without blocking workflow completion |
| Sí/No as toggle buttons | Mobile UX for binary readings; large tappable targets better than dropdown or text input |
| tipos_equipo dropdown in modal | Standardizes equipment type selection; ensures correct workflow templates load for new equipment |
