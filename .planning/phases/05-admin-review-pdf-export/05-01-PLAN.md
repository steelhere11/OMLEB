    **src/app/admin/reportes/[reporteId]/page.tsx:**
    - Server component that receives `params: Promise<{ reporteId: string }>` (Next.js 16 pattern)
    - Fetch the complete report with ALL related data in a single Supabase query:
      ```typescript
      const { data: reporte } = await supabase
        .from("reportes")
        .select(`
          *,
          folios(*, clientes(nombre, logo_url)),
          sucursales(nombre, numero, direccion),
          users:creado_por(nombre, rol),
          reporte_equipos(
            *,
            equipos(numero_etiqueta, marca, modelo, numero_serie, tipo_equipo),
            reporte_pasos(*, plantillas_pasos(nombre, procedimiento), fallas_correctivas(nombre, diagnostico))
          ),
          reporte_fotos(*),
          reporte_materiales(*)
        `)
        .eq("id", reporteId)
        .single();
      ```
    - Also fetch folio_asignados for team members: query `folio_asignados` joined with `users(nombre, rol)` where `folio_id` matches
    - Handle not found: if no reporte, show "Reporte no encontrado" with back link
    - Render a back link "← Reportes" to `/admin/reportes`
    - Pass all data to `<ReportDetail>` client component

    **src/app/admin/reportes/[reporteId]/report-detail.tsx:**

    - **Info section**: Card with grid of info rows (same style as research example):

      - Direccion: sucursal.direccion
      - Cliente: nombre
      - Problema reportado: folio.descripcion_problema
      - Equipo de trabajo: comma-separated team member names with roles
      - Fecha: formatted date
      - Estatus: status badge
    - **Equipment entries section**: Header "Equipos Atendidos ({count})"

        - Photos for this equipment: filter reporte_fotos where equipo_id matches. Display as a responsive grid (grid-cols-2 md:grid-cols-3 gap-3).
