  <name>Task 1: Admin server actions for edit and approve</name>
  <files>
    src/app/actions/reportes.ts
  </files>
  <action>
    Add three new server actions to the EXISTING `src/app/actions/reportes.ts` file. All three must check `user.app_metadata?.rol === "admin"` and return `{ error: "No autorizado" }` if not admin. Add these BELOW the existing functions in the file.

    **1. adminUpdateEquipmentEntry(entryId: string, _prevState: ActionState | null, formData: FormData): Promise&lt;ActionState&gt;**
    - Auth check: get user, verify admin role
    - Parse formData with existing `reporteEquipoSchema` from `@/lib/validations/reportes`
    - On validation failure, return `{ fieldErrors }` using `z.flattenError`
    - Prepare data: empty strings become null for diagnostico, trabajo_realizado, observaciones
    - Execute `.update(data).eq("id", entryId)` on `reporte_equipos`
    - On success: `revalidatePath("/admin/reportes")` and return `{ success: true, message: "Guardado" }`
    - NOTE: This does NOT update equipo_id because admin edits content, not which equipment. The form only allows editing tipo_trabajo, diagnostico, trabajo_realizado, observaciones.

    **2. adminSaveMaterials(reporteId: string, materials: { cantidad: number; unidad: string; descripcion: string }[]): Promise&lt;ActionState&gt;**
    - Auth check: get user, verify admin role
    - Validate each material with existing `reporteMaterialSchema`
    - Use same delete-all + re-insert pattern as existing `saveMaterials` function
    - Delete all `reporte_materiales` where `reporte_id = reporteId`, then insert new rows
    - `revalidatePath("/admin/reportes")` and return `{ success: true, message: "Materiales guardados" }`

    **3. approveReport(reporteId: string): Promise&lt;ActionState&gt;**
    - Auth check: get user, verify admin role
    - Execute `.update({ finalizado_por_admin: true }).eq("id", reporteId)` on `reportes`
    - `revalidatePath("/admin/reportes")` and return `{ success: true, message: "Reporte aprobado" }`
    - Do NOT change the report's estatus -- approval is independent of status. Admin sets `finalizado_por_admin = true`.

    Import additions: No new imports needed beyond what already exists in the file.
  </action>
  <verify>
    Run `npx next build --webpack` to verify no TypeScript errors. Verify the new functions are exported from the file.
  </verify>
  <done>
    Three new admin server actions exist: adminUpdateEquipmentEntry (with Zod validation), adminSaveMaterials (delete-all + re-insert), approveReport (sets finalizado_por_admin). All check admin role.
  </done>
