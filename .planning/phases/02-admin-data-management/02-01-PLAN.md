  <action>
    Follow the exact patterns from auth.ts (server actions) and usuarios/ pages (list + form).

    1. Create `src/app/actions/clientes.ts` with three server actions:

       **createCliente(_prevState: ActionState | null, formData: FormData): Promise&lt;ActionState&gt;**
       - Verify admin role via `supabase.auth.getUser()` + `app_metadata.rol === "admin"`
       - Extract nombre from formData, validate with clienteSchema.safeParse()
       - On validation failure: return `{ fieldErrors: z.flattenError(result.error).fieldErrors }`
       - Handle logo file upload: get "logo" from formData as File, check size <= 2MB, check type is jpeg/png/webp
       - Upload via `createAdminClient().storage.from("clientes").upload(...)` with path `logos/${crypto.randomUUID()}.${ext}`
       - Get public URL via `.getPublicUrl(filePath)`
       - Insert into clientes table: `{ nombre: result.data.nombre, logo_url: logoUrl }`
       - `revalidatePath("/admin/clientes")` then `redirect("/admin/clientes")`
       - Call redirect OUTSIDE try/catch (it throws internally)

       **updateCliente(id: string, _prevState: ActionState | null, formData: FormData): Promise&lt;ActionState&gt;**
       - Same auth check and validation
       - If new logo uploaded: delete old logo from storage first (fetch current client, parse old file path, `admin.storage.from("clientes").remove([oldPath])`), then upload new one
       - Update clientes table with new nombre and logo_url (only update logo_url if new logo uploaded)
       - `revalidatePath("/admin/clientes")` then `redirect("/admin/clientes")`

       IMPORTANT: The updateCliente action takes `id` as the first parameter (will be bound via `.bind(null, id)` in the edit form). The formData is the last parameter.

    2. Create `src/app/admin/clientes/page.tsx` (Server Component):
       - Fetch all clients: `supabase.from("clientes").select("*").order("created_at", { ascending: false })`
       - Cast to `Cliente[]`
       - Render a table with columns: Logo (small thumbnail or placeholder), Nombre, Fecha de creacion, Acciones (Editar link)
       - "Crear Cliente" button linking to /admin/clientes/nuevo
       - Empty state with appropriate message and CTA
       - Follow exact same dark theme styling as usuarios/page.tsx

    3. Create `src/app/admin/clientes/nuevo/page.tsx` ("use client"):
       - useActionState with createCliente
       - Form fields: nombre (Input, required), logo (file input with accept="image/jpeg,image/png,image/webp")
       - Style the file input to match admin dark theme (may need custom styling)
       - Back link to /admin/clientes
       - Error display (both field-level via state.fieldErrors and general via state.error)
       - Submit button with isPending loading state

    4. Create `src/app/admin/clientes/[id]/editar/page.tsx` (Server Component):
       - Fetch client by id: `supabase.from("clientes").select("*").eq("id", id).single()`
