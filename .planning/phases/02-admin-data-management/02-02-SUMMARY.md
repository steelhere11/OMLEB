## Decisions Made
- Equipment routes use `[sucursalId]` dynamic segments for branch scoping, matching the entity-within-entity relationship (not query params)
- Folio user reassignment uses delete-all + re-insert pattern -- simple and correct for small user sets (typically 2-5 per cuadrilla)
- Optional equipment fields (marca, modelo, numero_serie, tipo_equipo) use `.optional().or(z.literal(""))` to handle empty form strings from HTML inputs
- Empty strings converted to null on equipment update to keep database clean
- Folio status displayed as read-only badge in edit form -- status changes deferred to Phase 3 (technician reporting)
