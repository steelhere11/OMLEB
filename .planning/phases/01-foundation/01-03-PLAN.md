  <action>
    1. Install Serwist packages: `npm install @serwist/next` and `npm install -D serwist`

    2. Update `next.config.ts` to wrap with Serwist (from RESEARCH.md):
       ```typescript
       import withSerwistInit from "@serwist/next";
       const withSerwist = withSerwistInit({
         swSrc: "src/app/sw.ts",
         swDest: "public/sw.js",
         cacheOnNavigation: true,
         reloadOnOnline: true,
         disable: process.env.NODE_ENV === "development",
       });
       ```
       Wrap the existing nextConfig with `withSerwist(nextConfig)`.

    3. Update `package.json` build script to use `--webpack` flag:
       `"build": "next build --webpack"`
       This is REQUIRED because Serwist's Next.js plugin uses webpack. Without it, the service worker won't be generated (RESEARCH.md Pitfall 2).

    4. Create `src/app/sw.ts` -- Serwist service worker (from RESEARCH.md):
       - Import `defaultCache` from `@serwist/next/worker`
       - Import types `PrecacheEntry`, `SerwistGlobalConfig` from `serwist`
       - Declare global `__SW_MANIFEST`
       - Create Serwist instance with:
         - precacheEntries: self.__SW_MANIFEST
         - skipWaiting: true
         - clientsClaim: true
         - navigationPreload: true
         - runtimeCaching: defaultCache
         - fallbacks: offline entry pointing to "/~offline" for document requests
       - Call `serwist.addEventListeners()`

    5. Create `src/app/manifest.ts` -- PWA manifest using Next.js file convention (from RESEARCH.md):
       - Type: `MetadataRoute.Manifest`
       - name: "OMLEB"
       - short_name: "OMLEB"
       - description: "Reportes diarios de mantenimiento HVAC"
       - start_url: "/"
       - display: "standalone"
       - background_color: "#ffffff"
       - theme_color: "#3b82f6" (placeholder brand blue)
       - orientation: "portrait"
       - Icons: 192x192 (maskable) and 512x512

    6. Create `src/app/~offline/page.tsx` -- Offline fallback page:
       - Simple, clean page (no data fetching)
       - Centered content with WiFi-off icon (inline SVG)
       - Primary message: "Sin conexion a internet"
       - Secondary message: "Verifica tu conexion e intenta de nuevo. La aplicacion se recargara automaticamente cuando vuelvas a estar en linea."
       - Light background, readable text
       - No navigation elements (user may be anywhere in the app)
       - This page is precached by the service worker so it loads even when fully offline

    7. Create `src/components/shared/install-prompt.tsx` -- PWA install prompt:
       - Client component ('use client')
       - Listen for `beforeinstallprompt` event in useEffect
       - When event fires, show a dismissible banner at the bottom of the screen (above tab bar for technician):
         - "Instalar OMLEB" text

       - CONTEXT decision: show when browser detects installability, not forced

    8. Add the InstallPrompt component to the root layout (`src/app/layout.tsx`) so it shows on any page when the app is installable.

       - These are placeholders until the real OMLEB logo is provided.

    IMPORTANT: The service worker is DISABLED in development (`process.env.NODE_ENV === "development"`). It only activates in production builds. Testing PWA install requires `npm run build && npm run start`.
