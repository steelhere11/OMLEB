  <action>
    1. Create `src/proxy.ts` (NOT middleware.ts -- Next.js 16 convention):
       - Export `async function proxy(request: NextRequest)` -- NOT `middleware`
       - Call `updateSession` from `src/lib/supabase/proxy.ts` to refresh auth tokens
       - After session refresh, use `supabase.auth.getUser()` (NOT getSession) to get current user
       - Implement role-based routing logic from RESEARCH.md Pattern 2:
         - Unauthenticated user hitting /admin/* (except /admin/login) -> redirect to /admin/login
         - Unauthenticated user hitting /tecnico/* -> redirect to /login
         - Authenticated user with role !== 'admin' hitting /admin/* -> redirect to /tecnico
         - Authenticated admin hitting /tecnico/* -> redirect to /admin
         - Authenticated user hitting /login or /admin/login -> redirect to role-appropriate dashboard
       - Export config.matcher that excludes static assets, images, and manifest paths:
         `'/((?!_next/static|_next/image|favicon.ico|icons/|manifest.webmanifest|sw.js|.*\\.(?:svg|png|jpg|jpeg|gif|webp|ico)$).*)'`
       - IMPORTANT: Read user role from `user.app_metadata?.rol` (not user_metadata)

    2. Create `src/app/actions/auth.ts` with three server actions:

       **login(prevState, formData)**:
       - Extract email and password from formData
       - Create server Supabase client
       - Call `supabase.auth.signInWithPassword({ email, password })`
       - On error, return `{ error: 'Correo o contrasena incorrectos' }` (generic Spanish message)
       - On success, read role from `data.user.app_metadata?.rol`
       - Redirect to `/admin` if admin, `/tecnico` if tecnico/ayudante
       - Use `redirect()` from next/navigation (throws, so call AFTER the Supabase response)

       **logout()**:
       - Create server Supabase client
       - Call `supabase.auth.signOut()`
       - Redirect to `/login`

       **createTechnicianAccount(prevState, formData)**:
       - Extract email, password, nombre, rol from formData
       - Create server client, verify caller is admin via `getUser()` + `app_metadata.rol === 'admin'`
       - If not admin, return `{ error: 'No autorizado' }`
       - Create admin client (service role)
       - Call `adminClient.auth.admin.createUser()` with:
         - email, password
         - email_confirm: true (skip email confirmation)
         - app_metadata: { rol } (tecnico or ayudante)
         - user_metadata: { nombre }
       - Insert into public.users table: id, email, nombre, rol
       - On success, return `{ success: true, message: 'Cuenta creada exitosamente' }`
       - On error, return `{ error: error.message }`
       - Use `'use server'` directive at top

    3. Create `src/app/auth/callback/route.ts`:
       - Handle GET request for PKCE auth callback flow
       - Extract `code` from searchParams
       - Exchange code for session using `supabase.auth.exchangeCodeForSession(code)`
       - Redirect based on user role (admin -> /admin, tecnico -> /tecnico)
       - On error, redirect to /login with error param

    4. Create shared UI components (minimal, reusable, Tailwind-styled):

       **src/components/ui/button.tsx**: A `Button` component with variants:
       - `variant`: 'primary' (brand-colored), 'secondary' (outline), 'danger' (red)
       - `size`: 'sm', 'md', 'lg' (lg = large touch targets for mobile, min 48px height)
       - `loading`: boolean prop that shows a spinner and disables
       - Full width option (`fullWidth` prop)
       - Uses Tailwind classes, no external UI library

       **src/components/ui/input.tsx**: An `Input` component:
       - Wraps native input with consistent Tailwind styling
       - `error` prop: string -- shows red border and error message below input in Spanish
       - Large enough for mobile touch (min 48px height)
       - Supports password toggle (show/hide eye icon) when type="password"

       **src/components/ui/label.tsx**: A `Label` component:
       - Wraps native label with consistent font size and color
       - Supports `required` visual indicator (asterisk)
  </action>
